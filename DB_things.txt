----------------------------------------------------------
таблица things

id           номер вещи
name         имя для файла imes.txt
is_inventory флаг: может лежать в инвентаре (1-да,0-нет)
is_auto      автоматически использовать при обнаружении в инвентаре (если = 1)
is_vehicle   для вещей-транспорта = 1
rotx         вращать в инвентаре по оси X
roty         вращать в инвентаре по оси Y
rotz         вращать в инвентаре по оси Z
inventx      начальный угол в инвентаре по оси X
inventy      начальный угол в инвентаре по оси Y
inventz      начальный угол в инвентаре по оси Z
zoom         масштаб в инвентаре
posx         угол на земле по X
posy         угол на земле по Y
posz         угол на земле по Z
height       коррекция высоты
invent_id    SAMP-идентификатор предмета в инвентаре
inworld_id   SAMP-идентификатор предмета в игровом мире
rotatable    объект, меняющий наклон на склонах местности
def_value    значение поля 'objects.value' при инициализации
type_id      тип вещи (thing_type.id)
is_consumble флаг 'расходный материал'
-----------------------------------------------------------

Флаг is_auto используется для соединения вещей. Т.е., если вещь не
лежит (не отображается) в рюкзаке, то при is_auto=1 она всё равно
участвует в сборке. Например, если игрок сидит в машине, то при
сборке частей автомобиля машина также учитывается, хотя и не лежит
в рюкзаке. Использование этой опции пока находится под вопросом.

Надо бы ещё учитывать разницу высоты между лежащей вещью и сидячим
игроком, а не только стоячим.



----------------------------------------------------------
таблица objects

id           идентификатор объекта в базе данных
thing_id     идентификатор вещи
next_id      идентификатор вещи при её первом создании
prev_id      предыдущий тип данной вещи
placex       место сохранения вещи в игровом мире по X
placey       место сохранения вещи в игровом мире по Y
placez       место сохранения вещи в игровом мире по Z
add_rotx     начальное вращение по X
add_roty     начальное вращение по Y
add_rotz     начальное вращение по Z
pl_owner_id  владелец players.id
th_owner_id  владелец objects.id
obj_id       идентификатор созданного в игре объекта
dot_id       id места появления, если объект на нём лежит
is_dropped   показывает каким персонажем был выложен объект
unused       флаг отсутствия/присутствия объекта в игровом мире
value        значение, используемое каждым объектом по-своему
vector       расстояние до вещи от центра карты
----------------------------------------------------------

Поля is_dropped и dot_id используются для начальной инициализации
сервера. Таким образом все вещи, не имеющие хозяина и начальной
точки появления, при перезагрузке будут созданы в игровом мире
на прежнем месте.

В инвентаре указывается именно id объекта, а не id вещи!

Желательно вести кэш содержащихся в инвентаре объектов, а также
кэш типов вещей этих объектов. Под вопросом, т.к. требует много
памяти.

Функция, возвращающая id вещи, составленной из двух и более
объектов, выполняется только при полном соответствии выбранных
и составных частей. Для постепенного добавления объектов к неко-
торой вещи используется дополнительная функция, выполняющая
исключительно привязку одних объектов к другим (при условии, что
это возможно).

На будущее: при создании уничтоженной вещи используется поле
next_id, а не thing_id.
